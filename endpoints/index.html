# Endpoints

All endpoints are relative to the base url: https://plashiki.su/api

# Statistics

Receive statistics. Requires admin rights.

Special features:

  • Requires auth
  • Requires admin = true on current user

Throws

Error
 Description
NO_AUTH Not logged in
ACCESS_DENIED Flag not set

# Get raw statistics

GET
/v2/admin/statistics/

Retrieve raw statistic days for a given range.

Returns

StatisticsDay[]

For all days in a given range that have statistics info an item will end up there.Order is not guaranteed.

# Add donation/expense

POST
/v2/admin/budget

Add a donation or expense. Will automatically update associated Github file.

Parameters

Name Type Description
Body parameters (in JSON)
type "donation" ⁠|​⁠ "expense" Whether passed entity is a donation or an expense
date string Date of change in format: "DD.MM.YYYY в HH:MM".
amount number Amount of expense/donation in rubles (RUB)
comment string Comment for an entity. For a donation, if a user nickname is passed and user with such nickname exists, a donation will be added to their counter automatically. For expense, should be a human-readable expense destination

Returns

"OK"

# External batch addition

POST
/batch/add/translations

Batch addition for multiple translations at once as an external OAuth app. The only limit is bodyparser body limit (around 1mb), so with minification that's about 4k translations at once.
You can access extended number of parameters, including groupsstatus. Use status=pending to put items on pre-moderation, status=declined will be ignored.
Group from-app:[your-app-id] will always be added with your translations as a precaution.

Special features:

  • Requires server scope: batch:add:translation

Throws

Error
 Description
ACCESS_DENIED You did not pass a valid OAuth client_secret or your app does not have sufficient permissions
INVALID_TOKEN Given Token does not match any OAuth app.
