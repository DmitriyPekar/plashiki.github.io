# Translation

A single translation/submission (technically they are the same)

Fields

Name Type Description
id number Unique translation/submission ID.
target_id number Target media ID of the translation. Currently uses MAL IDs.
target_type string Target media type of the translation. Currently uses anime, manga like MAL.
part number Part number of the translation. Specifically, episode for anime and chapter for manga.
kind string Translation kind. Anime translation can be subbed and dubbed, manga translations can be scanlations and offficial, and both can be raw
lang string Translation language. Currently support en, ru, by, ua, jp, fr, de, cn, ko and other, but list may expand at some time.
author Translation author
uploader_id number ID of user that added this translation. null if added by Parser or External batch addition
status string Translation status. Differentiates submissions and actual translations. Only available in Moderation or Submission APIs.
url string URL to translation iframe. Is a https:// link (or http:// for external translations). If ?external=proto, then may be a ehttp:// or ehttps:// link, which means this translation is external and can't be displayed in a iframe. Remove preceding e to get actual link.
Checks:
  • Valid URL, protocol is one of: https
groups Translation grouping. Used to do batch operations with ease. Mainly used for Parsers like parser:[parser-uid] and External batch addition like from-app:[app-id]Only visible to moderators.
created_at Date Translation creation time
updated_at Date Translation last update time
uploader object Information about user that added this translation. Only present with ?needUploader param, only id, nickname, avatar fields are present.

# Parser

Parser is a piece of JS code + meta information for some actions, whose method of handling may change at all times. We use them to store (currently) 3 types of parsers -- Importers, Cleaners and Mappers.

Importers are scripts that run every so often (on a schedule) and which task is to grab some translations from some services and import them all to our DB. They should contain a function(){...} expression, where the returned function should return an async iterable (or compatible) of Translation

Cleaners are scripts that remove old and banned players from database. They should contain afunction(){} expression, where the returned function should return an async iterable (or compatible) of integers, ids of translations to be removed. In params, they have access to TypeORM Translation entity (ctx.params.Translation)

Mappers are scripts that grab external websites' media ids and try to map them with MAL ids, which are used by PlaShiki. They should contain a function(){...} expression, where the returned function should return ExternalServiceMappings or Promise<ExternalServiceMappings>

All of them receive a ParserContext object as an argument, which is described in types/ctx.ts at plashiki-parsers repository.

Fields

Name Type Description
uid string Uid of a parser. Is globally unique. Importers' uid start with importers/, Cleaners' uid start with cleaners/ and Mappers' uid start with mappers/.
provide string Dependencies of a parser. A parser can contain some dependencies, for example to abstract parsing logic of a single platform but keep different parsers for different authors. Nested dependencies are supported, circular are not
storage string List of used storage keys by a parser. This list may not be exhaustive, and is only used for parser dashboard. This DOES NOT affect parser's runtime permissions or whatever. May include SQL patterns, for example name:%
disabled boolean When a parser is disabled, it will not be started on schedule (in case of an Importer/Mapper) This WILL NOT disable importing it as a dependency.
public string When a parser is public, anyone can run it. Possible values: '' (empty string) = non-publictrue = publicN,M = public with rate-limit: N requests every M seconds
cri boolean Constantly running importer. If true then this importer will be running all the time while server is up in a worker thread. (however it wont be restarted after worker is dead, i.e. function returns). Only applicable to Importers.
code string JavaScript code
source string Original source code of a file. GZipped. Used only for pulling.
hash string MD5 hash of a parser. Calculated as follows:
md5(p.uid + '\n\n' + p.provide.sort().join(',') + '\n\n' + p.code).
Used to keep track of changed parsers when pushing/pulling them

# GetTranslationsParameters

Fields

Name Type Description
kind string Kind of translations to be returned.
Checks:
  • One of: sub, dub, scan, off, raw
lang string Language of translations to be returned.
Checks:
  • One of: en, ru, by, ua, jp, fr, de, cn, ko, other
hq boolean If true then only HQ translations will be returned
raw boolean If passed then items will not be sorted and classified, and returned as a raw array of objects
external string How to handle translations with external (non-embeddable) players. By default it will just return a link to it.
If ?external=page, then will return a special page that will ask user to open it in a new page
If ?external=proto, then external players will have a protocol ehttp:// or ehttps://, so you can handle them manually
Checks:
  • One of: page, proto
needUploader boolean If passed, all translation object will contain a uploader field. However, only id, nickname, avatar fields will be available.
fullAuthor boolean If passed, author will contain a TranslationAuthor object (only relevant when ?raw is passed)

# PatchUserBody

Fields

Name Type Description
admin boolean Toggles user's admin status. Only available to admins.
moderator boolean Toggles user's moderator status. Only available to admins.
trusted boolean Toggles user's trusted status. Only available to admins and moderators.
banned boolean Toggles user's banned status. Only available to admins and moderators.
nickname string Changes user's nickname. Value must be a spare nickname. Available for current user and for admins.
avatar object Changes user's avatar. Value must be a URL to image or null. Available for current user and for admins.
service string Changes user's user rate service. Value must exist in external_ids object. Only available for current user
